# 骨子 × 手法 マッピング

骨子（[ai-adoption-framework.md](./ai-adoption-framework.md)）と具体的手法（[ai-adoption-methods.md](./ai-adoption-methods.md)）の対応関係を整理したものです。

---

## 骨子の略称

| 骨子 | 略称 |
|---|---|
| ① AI活用を「文化」として根付かせる | ①文化 |
| ② AI活用の成果を「見える化」し、社内外に発信する | ②見える化 |
| ③ AI活用の「標準」を定め、品質と安全を担保する | ③標準 |
| ④ AI活用によって「差別化できる」組織になる | ④差別化 |
| ⑤ 継続的に「学び、アップデートし続ける」組織をつくる | ⑤継続学習 |
| ⑥ 全社横断の「ガバナンス」と推進体制を整える | ⑥ガバナンス |
| ⑦ 業務・プロセスを「AIネイティブ」な形に変革する | ⑦AIネイティブ |
| ⑧ AI活用の「効果・投資対効果」を測定・管理する | ⑧効果測定 |
| ⑨ 外部コミュニティへの「貢献とプレゼンス」を高める | ⑨外部プレゼンス |
| ⑩ 「責任あるAI利用」の姿勢を組織として示す | ⑩責任AI |
